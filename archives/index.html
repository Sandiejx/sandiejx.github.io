搭建HEXO博客

###什么是Hexo
Hexo是一种基于Node.js快速，简便，功能强大的博客框架，hexo用markdown来写文章，只需要几秒钟就可以生成静态网页。

###安装环境
Hexo博客系统是基于Node.js的，如果想要把博客搭建在免费的Github上，首先你需要在本地安装nodejs和Git。如果你已经安装了nodejs和Git，请跳过以下的1,2步骤。

###安装Node.js
安装Node.js最好的方式是用nvm
cURL:

$ curl https://raw.github.com/creationix/nvm/master/install.sh | sh

Wget:

$ wget -qO- https://raw.github.com/creationix/nvm/master/install.sh | sh

当nvm安装好之后，重新启动终端，用下面的命令来安装Node.js

$ nvm install stable

或者，也可以直接在Node.js官网选择适合的版本进行本地安装 。

###安装Git

  • windows ：下载并安装Git
  • Mac ：用Homebrew, MacPorts, installer等工具都可以安装
  • Linux(Ubuntu, Debian) ：sudo apt-get install git-core
  • Linux(Fedora, Red Hat, CentOS) ：sudo yum install git-core

###下载Hexo
当nodejs和Git都安装好之后，就可以用npm安装Hexo了:

$ npm install -g hexo-cli

###安装Hexo
当Hexo下载好之后，用下面的命令在文件夹folder下初始化:

$ hexo init folder  
$ cd folder 
$ npm install

初始化之后，本地项目里的文件目录结构如下：

.
├── _config.yml
├── package.json
├── scaffolds
├── source
|   ├── _drafts
|   └── _posts
└── themes

###简单介绍Hexo
_config.yml
配置文件，你可以在该文件里修改很多配置

package.json
依赖安装包数据，包括EJS，Stylus以及Markdown 是默认安装的，如果你暂时不需要你可以之后再安装。

package.json
{
  "name": "hexo-site",
  "version": "0.0.0",
  "private": true,
  "hexo": {
    "version": ""
  },
  "dependencies": {
    "hexo": "^3.0.0",
    "hexo-generator-archive": "^0.1.0",
    "hexo-generator-category": "^0.1.0",
    "hexo-generator-index": "^0.1.0",
    "hexo-generator-tag": "^0.1.0",
    "hexo-renderer-ejs": "^0.1.0",
    "hexo-renderer-stylus": "^0.2.0",
    "hexo-renderer-marked": "^0.2.4",
    "hexo-server": "^0.1.2"
  }
}

####源文件
source文件夹里是放博客的内容文件的。Hexo会忽略隐藏文件，以及那些以下划线开头命名的文件和文件夹，除了_post文件夹。渲染文件（比如markdown文件）会被处理并放到public文件夹下，而其他文件只会简单的被复制。

####博客主题
Themes文件夹里存放的是Hexo博客主题，Hexo会将网站内容和主题结合生成一个静态博客网站。

####Hexo主要命令
init（初始化） 

$ hexo init [folder]

初始化一个博客网站，如果没有文件夹，Hexo会在当前目录下新建一个博客文件夹

new（新建）

$ hexo new [layout] <title>

新建一篇文章，如果写了layout，Hexo会用_config.yml里default_layout设置的值，如果title里有空格，则会用引号括起来。

generate（生成）

$ hexo generate

生成静态文件
| 选项 | 描述 |
| :——– | :——– |
| -d –deploy | 当generate执行完之后，进行deploy部署 |
| -w –watch | watch(监听文件改变) |

publish（发布）

$ hexo publish [layout] <filename>

发布预览
server（服务）

$ hexo publish [layout] <filename>

跑一个本地服务器，默认的地址是http://localhost:4000/
| 选项 | 描述 |
| :——– | :——– |
| -p –port | 重写端口号 |
| -s –static | 只跑静态文件 |
| -l –log | 重写日志格式 |
deploy（发布）

$ hexo deploy

发布到线上Git
| 选项 | 描述 |
| :——– | :——– |
| -g –generate| 在发布之前先生成 |
